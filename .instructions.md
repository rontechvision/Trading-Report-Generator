# Claude Code Instructions - Trading Report Generator

## Professional Development Mode

This workspace is a **single-purpose Python skill**: generate a styled HTML
trading report from strategy backtest data, using design tokens cached from
a Figma file. Nothing else is in scope (no Telegram, no auto-trading, no
MCP orchestration) — see [CLAUDE.md](CLAUDE.md) for the boundary.

### Key Tools & Focus Areas
- **Language Server**: Pylance (Python analysis)
- **Templating**: Jinja2 (`skills/report_generator/templates/report.html.jinja2`)
- **Design System**: Figma REST API, tokens cached in `design_tokens.json`

### Task Priorities
1. **`report_generator` skill correctness** (`skill.py`, `cli.py`)
2. **Figma token extraction** (`mcp/figma_connector.py`,
   `skills/report_generator/figma_tokens_extractor.py`)
3. **Documentation stays in sync with actual file structure**

### Code Quality Standards
- Type hints for all functions
- Docstrings only where behavior isn't obvious from the signature
- Unit tests for data transformations (`skills/report_generator/tests/`)

### Development Workflow
1. Work in `skills/report_generator/skill.py`
2. Use `sample_data.json` for testing
3. Update `design_tokens.json` via `figma_cli.py fetch` (or edit by hand)
4. Run `python -m skills.report_generator.skill` to sanity-check output
